---
title: "Submit Change"
sidebarTitle: "Submit a Change"
description: "Submit changes done to the database (just like a commit being submited in git)"
---
<Warning>
   On this release version, avoid doing too many small commits on a graph and favorise bulk changes. It can lead to a loss in query performance after too many commits on a graph. This will be fixed in the next release. 
</Warning>

<Tabs>
  <Tab title="Python SDK">
    ```python
    # client.set_graph("my-graph")
    # client.checkout(my_change)
    client.query('CHANGE SUBMIT')
    client.checkout("main")
    ```
  </Tab>
  <Tab title="cURL">
    - Request

    ```bash
    # Submit a change to the graph you are working on
    curl "https://engines.turingdb.ai/sdk/query?graph=$GRAPH&change=$CHANGE" \
         -H "Authorization: Bearer $MY_API_KEY" \
         -H "Turing-Instance-Id: $MY_INSTANCE_ID" \
         -H "Accept: application/json" \
         -d 'CHANGE SUBMIT'
    ```

    - Response

    ```json
    {
      "header": {
        "column_names": [],
        "column_types": []
      },
      "data": [],
      "time": 0.055101
    }
    ```
  </Tab>
  <Tab title="Javascript">
    - Request

    ```jsx
    fetch(
    	`https://engines.turingdb.ai/sdk/query?graph=${GRAPH}&change=${CHANGE}`,
    	{
    		method: "POST",
    		headers: {
    			Authorization: `Bearer ${MY_API_KEY}`,
    			"Turing-Instance-Id": `${MY_INSTANCE_ID}`,
    			Accept: "application/json",
    		},
    		body: "CHANGE SUBMIT",
    	},
    );
    ```

    - Response

    ```json
    {
      "header": {
        "column_names": [],
        "column_types": []
      },
      "data": [],
      "time": 0.055101
    }
    ```
  </Tab>
</Tabs>
